## 2026-04-30 — Phase 1 full implementation

**What was added/changed:**

- `pyproject.toml` — all dependencies declared (Flask, SQLAlchemy, Alembic, APScheduler, Pydantic v2, Anthropic SDK, OpenAI SDK, gunicorn, watchdog, cryptography, python-frontmatter, jsonschema)
- `.env.example` — all env vars documented with descriptions and defaults
- `docker/Dockerfile` — Python 3.12-slim, pip install, volume, port 5000
- `docker/docker-compose.yml` — mounts `./data:/data`, env_file, restart policy
- `docker/entrypoint.sh` — runs `alembic upgrade head` then starts gunicorn
- `app/config.py` — `Config` and `TestingConfig` classes driven entirely from env vars
- `app/db.py` — SQLAlchemy engine + session factory; `get_session()` is a context manager
- `app/__init__.py` — Flask app factory: registers all blueprints, event bus, scheduler, CORS, static file route
- `app/scheduler.py` — APScheduler setup; `register_heartbeat()` and `remove_heartbeat()` for Phase 2
- `app/models/` — 8 ORM models: `Workspace`, `Swarm` (+ icon column), `Agent`, `Trigger`, `Event`, `Run`, `RunStep` (with composite index on run_id+sequence), `KnowledgeDocument`; all with `to_dict()` methods
- `app/api/` — 7 blueprints under `/api/v1`: health, workspaces (full CRUD + disk writes), swarms (full CRUD + hierarchy.json creation), agents (read-only), events (fire + list), triggers (full CRUD), runs (list + detail with steps)
- `app/core/event_bus.py` — in-process pub/sub with ThreadPoolExecutor; swappable behind same interface
- `app/core/llm.py` — `LLMClient` wrapping Anthropic and OpenAI; provider selected by `LLM_PROVIDER` env var
- `app/core/resolver.py` — scope-chain resolver (swarm → workspace → company); qualified references (`company/`, `workspace/`) override the chain; raises `ResolverError` on miss
- `alembic.ini` + `migrations/env.py` — wired to `Base.metadata`, `DATABASE_URL` env var override supported
- `migrations/versions/f955d8c8e028_phase1_initial_schema.py` — autogenerated migration creating all 8 tables + index
- `tests/conftest.py` — session-scoped `data_dir`, `app` (in-memory SQLite), and `client` fixtures
- `tests/test_resolver.py` — 12 tests covering all scope-walking, qualifier, and resource-type cases
- `tests/test_api.py` — 26 tests covering all Phase 1 endpoints (correct shapes, CRUD, 404s, validation errors)

**Bug fixes during implementation:**
- `Swarm` model was missing `icon` column — added
- `get_session()` was not a context manager — refactored to `@contextmanager`

**Test result:** 38/38 passed

**How to revert:** All files were new — delete the listed files. The migration can be rolled back with `alembic downgrade base`.
